## 用語ミニ辞典（この設計書に出てくる略語）

| 用語 | やさしい意味 |
|---|---|
| **リポジトリ / 直下** | プロジェクトのファイル一式を置く GitHub 上の「保管庫」。「直下」＝その一番浅い場所。 |
| **commit / push** | commit＝変更に名前をつけて記録。push＝それをクラウド（GitHub）へアップロード。 |
| **JSON** | データを書くためのテキスト形式。`words.json`＝単語データをこの形式で書いたファイル。 |
| **API** | プログラム同士が話すための「窓口」。Notion API＝Notion を読み書きする窓口。 |
| **同期（sync）** | Notion の最新の単語を取ってきて `words.json` に写し取る作業。 |
| **トークン** | サービスにアクセスするための「合鍵」。Notion トークン＝Notion を読み書きできる合鍵。 |
| **raw（ロー）** | GitHub のファイルの「中身だけ」を返す URL。飾りのない「生」のデータ。 |
| **キャッシュ / キャッシュバスト** | キャッシュ＝一度読んだデータを使い回す仕組み（古いまま残ることがある）。バスト＝取り直しを強制すること。 |
| **DEV / PROD** | DEV＝開発中（自分の PC で試す）。PROD＝本番（妹が使う公開版）。 |
| **config** | 「どの列が表・裏・検索対象か」だけを決める設定ファイル。単語データ本体とは別物。 |
| **サンプル / 見本データ** | 開発中の動作確認に使う少数の見本の単語（現在の5語）。 |
| **404** | 「ファイルが見つかりません」というエラー。 |
| **cron** | 「毎○分ごとに自動実行」するタイマー設定。 |
| **Secrets（金庫）** | GitHub が用意する、合鍵を安全に隠しておく金庫。コードには残らない。 |
| **カスタムGPT / GPT Action** | ChatGPT で自作する専用GPT。Action＝そのGPTに「外部の窓口を叩く能力」を足す機能。 |
| **OpenAPI** | API の使い方を決まった書式で書いた「取扱説明書」。GPT に窓口の叩き方を教えるのに使う。 |
| **enrichment** | 単語だけの入力から、意味・品詞・例文などを生成して情報を「肉付け」すること。 |

## 0. ゴールと全体像

〈**妹が GPT に単語を言う → GPT が意味などを考えて Notion に書く → 自動同期が Notion を読み `words.json` を更新 → アプリが最新を表示**〉という流れを成立させる。

現状は単語データがサンプル5語。M3 で「実データ（Notion 同期）」へ接続する。設計原則は DESIGN.md を踏襲：**列は決め打ちせず発見する／NULL（空欄）は正常／enrichment・同期・表示は疎結合**。

```
妹(ChatGPT Plus) ──単語──▶ カスタムGPT「単語登録くん」
                              │ 意味/品詞/関連語/例文/Tips を生成
                              ▼ Action で書き込み
                        Notion DB「英単語帳」（あなた所有・妹にゲスト共有）
                              │
        自動同期(15分ごと)──▶ Notion を読む → words.json に整えて保存(変化時のみ)
                              ▼
                     リポジトリ直下 data/words.json
                              │ アプリが raw から取得（再ビルド不要）
                              ▼
                     GitHub Pages の PWA（辞書 / クイズ）
```

## A. スコープと段階（staging）

### 成果物（4つ＋1ドキュメント）
1. **整える部分（正規化）** … Notion のプロパティを、アプリが読みやすい素直な形へ変換する純粋ロジック。**TDD の主対象**。
2. **同期の道具** … Notion API を呼び（全件取得）、1の変換を適用し、`data/words.json` を書き、変化時のみ commit する部分。
3. **自動実行の仕掛け** … 15分ごと＋手動で 2 を動かす仕組み（GitHub Actions）。
4. **アプリの取得元切替** … サンプルから、同期済み `data/words.json`（raw 取得）へ。
5. **enrichment 手順書**（`docs/gpt-action.md`）… Notion 準備／合鍵／GPT Action／指示文／共有／セキュリティ。コードでなく手順書。

### 実装順（依存が少ない順・各段階で検証可能）
- **Step 1**：整える部分を TDD（RED→GREEN）。ネット不要、机上で完結。品質の要。
- **Step 2**：同期の道具。あなたの Notion に手で数語入れ、ローカル実行で疎通確認。
- **Step 3**：自動実行＋Secrets 登録。Actions 上で実同期し、`data/words.json` が commit される。
- **Step 4**：アプリの取得元を raw へ切替。本番 PWA が実データを表示。
- **Step 5（並行可）**：`docs/gpt-action.md` を書き、あなたが GPT をセットアップ → 妹に共有。

Step 1〜2 はあなたの Notion 準備を待たずに着手できる。ユーザーは「設計確定後に Notion 整理から着手」を希望。整理は手順書（E-1）に沿って行い、Step 1〜2 の実装と並行してよい。

## B. データの置き場所と取得経路

「アプリが最新データをどこから取るか」の設計。ねらいは **データ更新＝アプリ再ビルド不要**。

- **本番データ**：リポジトリ直下 `data/words.json`（同期が commit）を、**raw の URL から取得**（鮮度重視、CORS `*`）。キャッシュは取り直しを強制（更新ボタンで `?t=時刻`）。
  - 代替案：Pages に同梱（毎回再デプロイが必要＝設計目標に反する／Pages 障害履歴あり→不採用）。jsDelivr（速いがキャッシュが強く鮮度が落ちる→フォールバック候補）。
- **開発中（DEV）**：アプリ同梱のサンプル（現在の5語）を見る。ネットなしで動き、確認が安定。
- **本番（PROD）**：raw から実データ。取得元の切替はプログラムの1か所のみ。
- **config（役割マップ）**：アプリ同梱のまま。滅多に変わらずコード隣接なので Notion には置かない。単語データとは別扱い。
- **初期化（bootstrap）**：切替時に 404 を避けるため、最初だけサンプルを `data/words.json` にコピーして commit。以降は同期が上書き。

## C. 同期のしくみ（取ってきて整える）

同期の道具がやること：

1. **取りこぼさず全部読む**：Notion は多いと小分けで返す（「続きはこちら」）。続きが無くなるまで繰り返し取る。
2. **どんな列でも壊れない**：決まった列を探すのではなく、Notion にある列を発見して順に処理する。列を足しても道具を直さずアプリまで流れる。
3. **複雑な形を素直な形へ整える**：種類ごとに変換ルールを決める。見たことのない種類はとりあえず文字化し「注意メモ」に残す（黙って壊れない）。
4. **空欄は正常**：値がない項目は空のまま記録。アプリがその行を静かに省く。
5. **変化時だけ保存**：15分ごとに動くが、中身が前回と同じなら保存し直さない。「実行時刻」は毎回変わるので比較から除外し、**単語の中身が本当に変わったときだけ**保存・記録する。
6. **品質は見本入出力で先に固める（TDD）**：「この入力ならこの結果」という見本を多数用意し、それを満たすよう作る。ネット・Notion なしで検証可。
7. **道具を2層に分ける**：〈Notion と通信する係〉と〈データを整える係〉に分け、要である「整える係」を見本で厳密にテストできるようにする。

---

## D. 自動で・定期的に動かす

- **自動実行**：GitHub の無料の自動実行に「15分ごとに同期の道具を動かす」を登録。あなたは何もしなくても裏で更新される。
- **タイミング**：定期15分ごと（妹の入力から最大15分で反映）＋「今すぐ動かす」手動ボタン。
- **合鍵の預け方**：Notion の合鍵は**公開ファイルに書かず**、GitHub の金庫（Secrets）に隠す。動作時だけ取り出し、コードに残さない。預けるのは「合鍵」と「どの Notion DB を見るかの住所（識別番号）」の2つ。
- **書き込み許可**：自動実行に「保管庫へ書き込んでよい」許可を明示的に付ける（更新結果を残すため）。
- **60日ルール（小さな注意点）**：保管庫に60日間まったく動きがないと定期実行が一時停止される仕様。長期間まったく変化がない場合に備え「たまに手動ボタン」で回避できる。
- **アプリの再公開ではない**：同期が触るのは単語データのファイルだけ。アプリ本体は作り直さない（B の分離のとおり）。

## E. Notion 準備と「単語登録くん」（enrichment 方式A）

`docs/gpt-action.md` に手順としてまとめる。難しい説明書（OpenAPI）と GPT 指示文は、貼れる形で用意する。

### E-1. Notion に「単語をためる箱」を作る（＝Notion 整理の実体）
「英単語帳」データベースを1つ作り、次の9列を用意する（あとから増やしてよい設計）。

| 列の名前 | 種類 | 役割 |
|---|---|---|
| 英単語 | 見出し（必須） | カードの表・単語そのもの |
| 意味 | 文章 | カードの裏・日本語の意味 |
| 品詞 | 選択肢を1つ | 名詞/動詞/形容詞… |
| 関連語 | タグを複数 | 似た語・関連語 |
| イディオム | 文章（任意） | 熟語（空でOK） |
| 例文 | 文章 | 使い方の例 |
| Tips | 文章 | 覚え方・語源メモ |
| レベル | 数字 | 難易度（クイズの絞り込みに使う） |
| タグ | タグを複数 | 分類（クイズの絞り込みに使う） |

### E-2. 合鍵を作って GitHub の金庫に預ける
- Notion で「外部から触るための合鍵（トークン）」を発行し、**英単語帳の DB だけ**に権限を絞る（最小限）。
- GitHub の金庫に「合鍵」と「英単語帳の住所（識別番号）」を登録。→ 自動同期が Notion を読めるようになる。

### E-3. 妹が使う「単語登録くん」を作る
- あなたが ChatGPT でカスタムGPTを1つ作る。
- そのGPTに「Notion に書き込む窓口を叩く能力（Action）」を追加。書き込み用の合鍵と、窓口の使い方（説明書）を渡す。
- GPTへの指示：「英単語を受け取ったら意味・品詞・関連語・例文・Tips を生成し、Notion に1行登録。わからない項目は空のままでよい」。説明書と指示文は本手順書に用意する。

### E-4. 妹に渡す
- GPTを**限定リンク**で妹にだけ共有。
- （任意）Notion の DB を妹に**ゲスト共有**すれば、iPad で直接閲覧・編集も可能（所有はあなたのまま）。

### E-5. 安全のための約束事
- GPTのリンクを知る人はその Notion に書ける → **家族内だけの私的リンク**運用。
- 合鍵は英単語帳だけに権限を絞る。
- 漏洩時は Notion で合鍵を作り直せば古い合鍵は即失効（ローテーション）。

**合鍵の数**：読む用（同期）と書く用（GPT）で分けると、より安全。ただし管理が増えるので、**まずは1つで始め、慣れたら2つに分ける**を推奨（どちらでも動く）。

## F. 品質の確かめ方と「完成」条件（受け入れ基準）

**確かめ方**：整える部分＝机上の見本入出力で検証／通信部分＝あなたの Notion に数語入れて実行確認／アプリ＝ブラウザで実データ表示を目視。

**完成条件（段階ごと）**
1. **整える部分**：用意した見本が全部通る。空欄・複数タグ・数値・未知の種類も設計どおり整う。
2. **同期の道具**：Notion の数語が正しい単語データになる。2回連続で動かし、変化がなければ保存し直さない。
3. **自動実行**：15分ごと＋手動で動き、更新結果が保管庫に残る。合鍵はコードに一切出ない。
4. **アプリ切替**：本番アプリが実データを表示。〈単語追加 →(最大15分 or 手動)→ 更新で反映〉が通る。エラー表示ゼロ。
5. **手順書**：あなたが手順書だけを見て Notion 準備〜GPT セットアップ〜妹共有まで迷わず一通りできる。

## 付録：技術詳細（実装者向け）

> 自然言語パート（A〜F）を、実装に落とすための具体。用語・ファイル名を含む。

### G-1. 正規化（Notion プロパティ → JSON）
- **フィールド発見**：`GET /v1/databases/{id}`（データベース取得）で得た `properties` を**正規の列一覧・順序**とし、各ページをそれに沿って正規化する（＝新規追加した空列も `meta.fields` に載る／順序が安定）。取得ページのプロパティ和集合はフォールバックの安全網。
- **型変換ルール（DESIGN §4.3 準拠）**：

  | Notion 型 | JSON 表現 | 未入力(NULL) |
  |---|---|---|
  | `title`,`rich_text` | 連結プレーンテキスト(string) | `null` |
  | `number` | number | `null` |
  | `select`,`status` | 選択名(string) | `null` |
  | `multi_select` | 選択名の配列(string[]) | `[]` |
  | `checkbox` | boolean | `false` |
  | `date` | ISO文字列(start) | `null` |
  | `url`,`email`,`phone_number` | string | `null` |
  | `people` | 名前の配列(string[]) | `[]` |
  | `files` | URL配列(string[]) | `[]` |
  | `formula`,`rollup` | 内部値を解決 | `null` |
  | `relation` | ページID/タイトル配列（v1は任意） | `[]` |
  | 未知/新型 | best-effort 文字列化 ＋ `meta.warnings` に記録 | `null` |

- **出力形**：DESIGN §4.2 の `words.json`（`meta.generatedAt/source/count/fields/warnings` ＋ `words[]`、各語に `id` と `_updated`）。
- **純粋関数として分離**：`normalize(page, schema) → wordRecord`、`buildFields(schema) → FieldMeta[]` などを IO から切り離し、vitest でテスト（`environment: node`、RED→GREEN）。テスト配置はスクリプト側に置き、vitest の `include` を拡張して拾わせる（現状 `src/**/*.test.ts`）。fixture は実 Notion API レスポンスの縮小版を使う。

### G-2. 同期スクリプト（IO 層）
- `scripts/sync-notion.mjs`：Secrets（`NOTION_TOKEN`,`NOTION_DATABASE_ID`）で DB を query。`start_cursor`/`has_more` でページネーション全件取得。各ページを G-1 で正規化。`data/words.json` を生成。
- **差分検出**：既存 `data/words.json` を読み、**`meta.generatedAt` を除外**して内容比較（正規化後の words＋fields）。同一なら書き込み・commit をスキップ。差分時のみ書き込み＋commit。
- Node 標準（`fetch` 同梱の Node バージョン前提）。Notion SDK は任意（依存を増やさないなら素の fetch でも可）。

### G-3. ワークフロー
- `.github/workflows/sync.yml`：`on: schedule: cron '*/15 * * * *'` ＋ `workflow_dispatch`。`permissions: contents: write`。Secrets 参照。差分時のみ commit＆push。
- **deploy.yml との関係**：`deploy.yml` は `paths-ignore: data/**` 済み → データ commit では再デプロイされない（設計どおり）。Step 4 のコード変更（取得元切替）は通常どおりデプロイをトリガする。
- 60日休眠注意（public repo の schedule 自動無効化）を README/コメントに明記。

### G-4. アプリ取得元切替
- `src/lib/data.ts`：`WORDS_URL` を DEV/PROD で分岐。
  - PROD：`https://raw.githubusercontent.com/kajisaden/hiyolingo/main/data/words.json`（＋`?t=` バスト）
  - DEV：`${import.meta.env.BASE_URL}data/words.json`（同梱サンプル）
- `CONFIG_URL` は現状維持（`${BASE_URL}data/config.json`、同梱）。
- bootstrap：初回に `public/data/words.json` を `data/words.json` にコピーして commit（切替後 404 回避）。

### G-5. `docs/gpt-action.md`（手順書の内容）
- Notion DB スキーマ（E-1 の9列・型）／インテグレーション作成・DB 共有・トークン取得／GitHub Secrets 登録手順。
- カスタムGPT の Action：サーバ `https://api.notion.com`、認証 `Authorization: Bearer <token>`、ヘッダ `Notion-Version: 2022-06-28`、操作 `POST /v1/pages`（必要に応じ query/patch）。**OpenAPI をそのまま貼れる形で記載**。
- GPT 指示文（プロパティ名マッピング、不明は空、重複配慮）。
- 妹への限定リンク共有・DB ゲスト共有・セキュリティ（私的リンク運用、権限最小化、漏洩時ローテーション）。

## 前提・未決事項

- **前提**：あなた＝ChatGPT Plus＋Claude Max、妹＝ChatGPT Plus（確認済み）。Notion 所有＝あなた、妹にゲスト共有。リポジトリ＝public `kajisaden/hiyolingo`、base path `/hiyolingo/`。
- **未決（実装時に確定）**：合鍵を1本/2本のどちらで始めるか（推奨：1本→後で分割）。フォールバックに jsDelivr を入れるか（当面 raw のみ）。テスト配置の具体（scripts 側 or src 側）と vitest include の書き方。
- **cron 頻度**：15分（最小5分まで短縮可）。
